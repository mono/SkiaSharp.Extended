Pluggable tile bitmap cache for Deep Zoom rendering. Implement this interface to provide custom caching strategies (in-memory, disk, tiered, etc.).

public interface ISKDeepZoomTileCache : IDisposable

Properties

Count

Number of tiles currently in the cache.

Methods

Clear()

Clears all cached tiles.

Contains(SKDeepZoomTileId)

Returns true if the tile is cached.

FlushEvicted()

Disposes bitmaps evicted since the last call. Call once per render frame on the UI thread before drawing to safely release GPU-bound bitmaps.

Put(SKDeepZoomTileId, SKBitmap?)

Stores a tile bitmap synchronously. Use this for simple in-memory caches.

PutAsync(SKDeepZoomTileId, SKBitmap?, CancellationToken)

Stores a tile bitmap. Implementations may apply delays or tiered writes. Called from a background thread; the ct should be respected.

Remove(SKDeepZoomTileId)

Removes a specific tile from the cache.

TryGet(SKDeepZoomTileId, out SKBitmap?)

Tries to retrieve a cached tile bitmap.

TryGetAsync(SKDeepZoomTileId, CancellationToken)

Asynchronously tries to retrieve a cached tile bitmap. Use this for cache tiers that involve I/O (browser storage, disk, etc.). A null return means cache miss -- the caller should then fetch from network.
