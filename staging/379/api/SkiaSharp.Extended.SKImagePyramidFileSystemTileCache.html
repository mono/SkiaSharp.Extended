A two-tier tile cache: LRU memory (L1) + local filesystem (L2). Tiles are stored as raw encoded bytes on disk — no re-encoding needed. The filesystem tier survives app restarts and reduces network usage.

Remarks

Recommended base paths by platform:

  • MAUI: FileSystem.CacheDirectory
  • Desktop .NET: Path.Combine(Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData), "SkiaSharpCache")
  • Fallback: Path.GetTempPath()
Blazor WebAssembly has no real filesystem; use SKImagePyramidBrowserStorageTileCache from SkiaSharp.Extended.UI.Blazor instead.

Set ActiveSourceId (the controller does this automatically on Load()) so tiles from different image sources are stored in separate subdirectories.

DefaultExpiry

Default tile expiry when none is specified.

Properties

ActiveSourceId

The identifier of the currently active image source. Set by the controller when a new source is loaded via Load(). I/O-backed caches (e.g. filesystem, browser storage) use this to namespace tiles per source. In-memory caches may ignore this value.

Count

Number of tiles currently in the cache.

Expiry

Maximum age for cached tiles. Files older than this are treated as a miss and deleted on next access. Can be updated at runtime (e.g. by the controller when loading a source with a shorter CacheExpiry).

Tries to retrieve a cached tile synchronously. Only checks in-memory storage.

TryGetAsync(SKImagePyramidTileId, CancellationToken)

Asynchronously tries to retrieve a cached tile, including any L2 storage (disk, browser). A null return means cache miss — the caller should then fetch from network.
